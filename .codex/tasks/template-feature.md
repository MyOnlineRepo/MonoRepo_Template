# Feature Task

## Ziel

Diese Vorlage dient zur Beschreibung einer neuen Funktion oder einer funktionalen Erweiterung in pragmatischer Form.

## Kurzbeschreibung

- Was soll ergänzt oder neu gebaut werden?
- Welcher Bereich ist betroffen?
- Welcher Nutzen wird erwartet?

## Betroffener Bereich

- Backend:
- Frontend:
- Infrastruktur:
- Sonstiges:

## Bestehender Pfad

- Welche bestehende Struktur, welches Modul oder welche Datei ist der richtige Startpunkt?
- Welche bestehenden Patterns oder Utilities sollen bevorzugt weiterverwendet werden?

## Umsetzung

- Welche minimale Änderung ist sinnvoll?
- Welche Teile gehören ausdrücklich nicht zum Scope?
- Welche Annahmen werden getroffen?

## Validierung

- Welche Checks sind sinnvoll?
- Welche Tests sollen ergänzt oder angepasst werden?
- Welche manuelle Prüfung ist ausreichend?

## Risiken und offene Punkte

- Welche Risiken gibt es?
- Was ist noch offen?
- Was sollte bewusst erst später entschieden werden?

## Abschluss

Zum Abschluss kurz festhalten:

- was umgesetzt wurde
- welche Dateien betroffen waren
- welche Checks ausgeführt wurden
- welche Annahmen oder offenen Risiken bleiben
